# ARPG 项目 · 热更 & 打包工作流手册

> 技术栈：HybridCLR（C# 代码热更）+ YooAsset（资源分发/版本化增量更新）
> 适用结构：2026-06 程序集边界重构后（AOT 引导 + 单一热更程序集）
> 维护提示：本手册描述"为什么"和"按什么顺序做"，具体数值（CDN 地址、保留版本数）以工程里实际配置为准。

---

## 1. 热更总览

### 1.1 程序集边界（谁能热更）

| 程序集 | 位置 | 性质 | 能否热更 |
|--------|------|------|---------|
| `Assembly-CSharp` | `Assets/Scripts/AOT/Main.cs` | AOT 引导 | ❌ |
| `Bootstrap` | `Assets/Scripts/AOT/Bootstrap/`（IGameLauncher、BootPanel） | AOT 契约层 | ❌ |
| `HotUpdate` | `Assets/Scripts/HotUpdate/`（全部游戏逻辑） | **热更** | ✅ |

- 依赖方向铁律：**热更 → AOT 合法；AOT → 热更非法**。所以 `HotUpdate` 引用 `Bootstrap`，但 AOT 侧任何程序集都不引用 `HotUpdate`。
- `HotUpdate.asmdef`：`autoReferenced = false`（防止默认程序集静态链接它）；references 含 `Bootstrap` + 各依赖包。
- `HybridCLRSettings.asset` 的 `hotUpdateAssemblyDefinitions` 指向 `HotUpdate`（guid `01644312…`）。
- 真机加载的热更 DLL：`Assets/AB/HotDll/HotUpdate.dll.bytes`。

### 1.2 启动流程

```
Boot.unity（唯一内置场景，只含 Main + BootPanel）
 └─ Main.Start → YooAssets 初始化 → 版本检查 → 清单更新 → 下载
 └─ LoadAndEnterGame：
      #if UNITY_EDITOR  从已加载程序集反射取 ARPG.GameLauncher（不加载 bytes）
      #else             补充 AOT 元数据 → Assembly.Load(HotUpdate.dll.bytes) → 反射取类型
 └─ (IGameLauncher)launcher.Launch()   ← 唯一一次 AOT→热更握手
      └─ GameLauncher(热更内)：GameArchitecture.Interface.SendCommand<TransitionToBeginSceneCommand>()
           └─ 之后全部逻辑都在热更程序集内，互相普通方法调用，无需再跨界
```

> 核心理解：**反射只用于"点火"一次。点着后整个游戏都跑在热更程序集里，所以改任意逻辑、下发新 DLL 即可生效。**

## 2. 关键目录与文件速查

| 路径 | 作用 |
|------|------|
| `Assets/Scripts/AOT/` | 不可热更：Main + Bootstrap |
| `Assets/Scripts/HotUpdate/` | 可热更：全部游戏逻辑 |
| `Assets/AB/` | YooAsset 资源源目录（Character/Enemy/Data/UI/Scene/FX/HotDll…） |
| `Assets/AB/HotDll/HotUpdate.dll.bytes` | 热更逻辑 DLL（下发） |
| `Assets/AB/HotDll/*.dll.bytes`（其余 8 个） | **AOT 补充元数据** DLL（非逻辑，给泛型用） |
| `Assets/HybridCLRGenerate/AOTGenericReferences.cs` | 自动生成：`PatchedAOTAssemblyList`（要补元数据的程序集）+ 泛型记录 |
| `Assets/HybridCLRGenerate/link.xml` | 自动生成：防裁剪保留名单 |
| `HybridCLRData/HotUpdateDlls/<平台>/` | 编译出的热更 DLL 原件 |
| `HybridCLRData/AssembliesPostIl2CppStrip/<平台>/` | 出包后生成的裁剪版 AOT 程序集（元数据来源） |
| `Bundles/<平台>/DefaultPackage/<版本>/` | 每次打包的版本快照（部署到 CDN 的就是它） |
| `Bundles/<平台>/DefaultPackage/OutputCache/` | 增量构建缓存（**勿删**） |

## 3. 工作流 A：基线出包（AOT 侧有变更时）

> 触发条件：改了 `Main.cs`/`Bootstrap`，或首次/大改后需要一份新的安装包基线。

1. **编译通过**：回 Unity，Console 零报错。
2. **`HybridCLR/Generate/All`**：生成 link.xml、AOTGenericReferences、方法桥。
3. **对齐 `aotMetadataDlls`**：打开 `Boot.unity` → 选 Main → Inspector 的 `aotMetadataDlls` 列表与 `AOTGenericReferences.PatchedAOTAssemblyList` 保持一致（**不含 HotUpdate/旧 GameFramework，只含 BCL/Unity 那几个**）。⚠️ 必须在出包前改，因为它存在 Boot 场景里、会被打进安装包。
4. **`Build Player`**：出安装包；同时生成 `AssembliesPostIl2CppStrip`。
5. **`Tools/HybridCLR工作流/复制全部 DLL`**：把热更 DLL（`HotUpdate.dll.bytes`）+ AOT 元数据 DLL 拷进 `Assets/AB/HotDll/`。
6. **打 YooAsset bundle**（Builder 窗口）。
7. **部署**：用 `Tools/AB包资源部署工具` 把版本快照增量上传到 CDN（`CDN/PC/ARPGv1.0/`）。

---

## 4. 工作流 B：日常热更（只改逻辑，最常用）

> 触发条件：只改了 `HotUpdate/` 里的游戏逻辑（战斗、数值、UI、AI、Command/Query…）。**不用重出安装包。**

1. 改 `HotUpdate/` 代码。
2. **`HybridCLR/Generate/All`**（或至少 `CompileDll`）。
3. **`Tools/HybridCLR工作流/仅复制热更 DLL`**（产物 = `HotUpdate.dll.bytes`）。
4. 打 YooAsset bundle。
5. 用部署工具上传到 CDN。
6. 客户端**不重装**，重启走更新流程 → 改动生效。

---

## 5. 工作流 C：什么时候"必须"重出 AOT 安装包

只有 AOT 侧必须变化时才需要，分两种：

1. **改了 AOT 代码**（`Main.cs` / `Bootstrap`）。
2. **热更代码用到了"基线没烤过的值类型泛型"**（如新的 `List<某Struct>`、`Dictionary<int,某Enum>`）：
   - 真机会报 `MissingMethodException` / "AOT generic method … not instantiated"。
   - 处理：`Generate/All`（把该泛型写进 link.xml）→ 重出安装包 → 纳入新基线。

> 经验法则：
> - **引用类型泛型**（class）→ 不用重出包，下发 DLL 即可（共享代码 + 补充元数据搞定）。
> - **值类型泛型**且基线未覆盖 → 重出一次包。
> - 要不要重出，**以真机是否报"泛型未实例化"为准**，不是"加泛型就必须"。
> - 因此：首次出包前尽量把游戏跑全（战斗/UI/存档/Boss…），让 HybridCLR 把用到的泛型都扫进基线，之后绝大多数热更都不用重出包。

## 6. 编辑器内测试说明

- 本项目编辑器里走的是 **YooAsset HostPlayMode（联机模式）**，所以编辑器 Play 也会去 **CDN** 拉 version/manifest/bundle。
  - ⚠️ **CDN 必须可达**（服务器开着 + Main 的 `defaultHostServer` 地址正确 + 上面摆着已构建的 bundle），否则会卡在"获取资源版本失败：Request timeout"。
  - 局域网 IP 变动会导致地址失效；建议给资源服务器固定 IP 或用主机名。
- 编辑器**不会真的 `Assembly.Load` 热更 DLL**（那样会和 Unity 已原生加载的同名类型冲突）。`Main.cs` 的 `#if UNITY_EDITOR` 分支改为从已加载程序集反射取 `GameLauncher`。所以**编辑器里跑的是当前编译的代码**，真正"加载 DLL 字节"的完整热更只发生在真机/IL2CPP 包。
- 如果未来想让编辑器不依赖 CDN 调试，可改用 YooAsset 的 EditorSimulateMode（本项目当前未采用，保持 HostPlayMode）。

## 8. HybridCLR 使用注意点

1. **依赖方向**：热更可引用 AOT，AOT 不可引用热更。跨界只在启动那一次（`IGameLauncher`）。游戏中途不需要事件桥——同程序集内直接调用即可（原 Boss 桥接事件已因此删除）。
2. **编辑器禁止二次加载热更 DLL**：必须有 `#if UNITY_EDITOR` 分支走反射，否则类型分裂崩溃。
3. **AOT 泛型元数据是头号坑**：
   - `aotMetadataDlls`（Boot 场景 Main 上）必须与 `PatchedAOTAssemblyList` 一致；漏了 → 真机泛型崩。
   - 改完热更逻辑记住链条：**改代码 → Generate/All → 同步 aotMetadataDlls（如有变化）→ 复制 DLL**。
4. **值类型泛型可能要重出包**（见工作流 C）。
5. **热更 MonoBehaviour 的前提**：引用它的 prefab/场景必须经 YooAsset AB 在 DLL 加载**之后**载入（本项目满足：Boot 场景外的资源全走 AB）。
6. **AOT 边界要最小**：只有 Boot 场景引用的类型（Main、BootPanel）必须在 AOT。**不要新增引用热更类型的内置场景/Resources 资产**，否则会把热更类型逼回 AOT。
7. **改了 asmdef 名/结构** → 必须重新 `Generate/All`（link.xml、AOTGenericReferences 按新名重生）。

## 9. YooAsset 使用注意点

1. **版本化增量更新**：每次打包 = 一个"版本快照目录"，历史默认全留（差量更新需要）。bundle 以内容哈希命名，跨版本相同内容自动去重。
2. **OutputCache vs 版本目录**：增量构建复用的是 `OutputCache`（**勿删**）；版本快照目录可清（用项目里的清理工具）。删旧版本快照不影响增量构建，也不影响运行。
3. **部署顺序**：先传新 bundle，**清单/version 文件最后传**——否则客户端可能拿到新清单但 bundle 还没传完。项目的部署工具已按此排序。
4. **CDN 必须提供**：`DefaultPackage.version` + 当前版本 manifest + 所有引用到的 `.bundle`。改了结构（如本次程序集改名）后，**必须重新打 bundle 并重新部署**，旧 CDN 内容会失效。
5. **HotDll 也是资源**：`HotUpdate.dll.bytes` + 8 个 AOT 元数据 DLL 都在 `Assets/AB/HotDll/`，**随 bundle 上 CDN**，真机启动时先下载、加载它们再跑热更。
6. **清服务器旧 bundle 要谨慎**：若线上仍有客户端停在旧版本、依赖那些 bundle，别删（部署工具的"清理过期 bundle"开关默认关，确认无旧客户端再开）。
7. **编辑器联机模式**：见第 6 节，需要 CDN 可达。

## 10. 常见报错对照表

| 现象 | 根因 | 处理 |
|------|------|------|
| `获取资源版本失败：Request timeout` | CDN 不可达 / 地址错 / bundle 未部署 | 开服务器、改对 `defaultHostServer`、重新打包部署 |
| `未找到 ARPG.GameLauncher` | 热更 DLL 没加载/没拷贝/文件名不对 | 确认 `复制热更 DLL` 后 HotDll 有 `HotUpdate.dll.bytes`，且 Main 加载路径一致 |
| 编辑器里类型重复 / 类型不一致 | 编辑器误用 `Assembly.Load` 加载了热更 DLL | 确认 `LoadAndEnterGame` 的 `#if UNITY_EDITOR` 反射分支在 |
| 真机 `MissingMethodException` / "generic method not instantiated" | 值类型泛型未烤进 AOT | `Generate/All` → 重出安装包（工作流 C） |
| 真机泛型相关崩、编辑器正常 | `aotMetadataDlls` 与 `PatchedAOTAssemblyList` 不一致 | 重新 Generate 并对齐列表、重拷元数据 DLL |
| prefab 上 Missing Script | 脚本 GUID 变了 | 本项目移动文件保留了 .meta，理论不会发生；若有则核对 .meta GUID |

## 11. 名词速查

- **AOT 程序集**：提前编译成原生、随安装包发布、不可改的程序集（Main、Bootstrap）。
- **热更程序集**：以 IL 字节下发、运行时由 HybridCLR 解释执行、可改的程序集（HotUpdate）。
- **AOT 补充元数据**：被裁剪掉的 AOT 泛型元数据，用未裁剪的 AOT 程序集 DLL 在启动时补回（`LoadMetadataForAOTAssembly`）。
- **PatchedAOTAssemblyList**：被热更代码泛型触及、因而需要补元数据的 AOT 程序集名单（≠ "被裁剪的程序集"，裁剪是普遍的）。
- **版本快照目录**：YooAsset 每次构建产出的、含全套 bundle + 该版本清单的目录，是部署到 CDN 的单位。
- **OutputCache**：YooAsset 增量构建的工作缓存。

---

## 一句话记忆

> **改逻辑** → Generate/All → 仅复制热更 DLL → 打包 → 部署 → 客户端重启生效（工作流 B）。
> **改 AOT / 撞上新值类型泛型** → 才需要重出安装包（工作流 C）。
> 编辑器测试要 CDN 在线；真机才是完整热更闭环。
